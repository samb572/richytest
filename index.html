Player Point Projection Formula

This document formalizes the logic for projecting a player's total points for an upcoming game. The core idea is to project various attempts and efficiencies, then combine them to estimate total points.

Core Idea

Projected Points = (Projected FGA * Projected FG%) + (Projected 3PA * Projected 3P%) + (Projected FTA * Projected FT%)

This formula refines how we arrive at those projected attempts and efficiencies by considering various player, opponent, and game-specific factors.

Formula Variables & Definitions

Let's use the following variables:

Player Specific (Baseline/Historical - preferably from the most recent full season or a significant sample size like last 30 games):

  • MPGplayer: Player's average Minutes Per Game
  • USG%player: Player's Usage Rate
  • FGA36: Player's average Field Goal Attempts per 36 minutes
  • FTA36: Player's average Free Throw Attempts per 36 minutes
  • FG%player: Player's Field Goal Percentage (overall)
  • 2P%player: Player's Two-Point Field Goal Percentage
  • 3P%player: Player's Three-Point Percentage
  • FT%player: Player's Free Throw Percentage
  • 3PArplayer: Player's Three-Point Attempt Rate (3PA / FGA)

Opponent Specific (Defensive Metrics - ideally recent, e.g., last 10 games or season-to-date):

  • DVPFGA_Pos: Opponent's DVP (Defense vs. Position) metric for Field Goal Attempts Allowed to the player's primary position, relative to the league average (e.g., if league avg is 100%, and they allow 105%, this is 1.05).
  • DVPFTA_Pos: Opponent's DVP for Free Throw Attempts Allowed to the player's primary position, relative to the league average.
  • PACEOpponent: Opponent's estimated possessions per game (Pace).
  • PACELeagueAvg: League average possessions per game.

Game Specific (Projection for the upcoming game):

  • MPproj: Player's Projected Minutes for the upcoming game. This is the most crucial manual input/estimation.
  • USG%adj: Any anticipated adjustment to the player's usage rate for this specific game (e.g., if a star teammate is out, this might be >1; if the player is returning from injury, it might be <1). Default = 1.0.

Step-by-Step Formula Derivation

Step 1: Calculate Game Pace Adjustment

The pace of the game dictates the total number of possessions, which in turn influences the number of offensive opportunities.

PaceAdj = \frac{PACE_{Opponent}}{PACE_{LeagueAvg}}

Step 2: Project Field Goal Attempts (FGA)

We'll start with the player's per-36 FGA, scale it to their projected minutes, adjust for their specific game usage, and then adjust for the opponent's DVP against attempts.

FGAproj = FGA36 \times \left( \frac{MP_{proj}}{36} \right) \times USG\%_{adj} \times DVP_{FGA\_Pos} \times PaceAdj

Step 3: Project Three-Point Attempts (3PA) and Two-Point Attempts (2PA)

Using the player's typical 3-point attempt rate.

3PAproj = FGAproj \times 3PArplayer
2PAproj = FGAproj - 3PAproj

Step 4: Project Free Throw Attempts (FTA)

Similar to FGA, we scale by minutes, adjust for usage, and DVP for free throws.

FTAproj = FTA36 \times \left( \frac{MP_{proj}}{36} \right) \times USG\%_{adj} \times DVP_{FTA\_Pos} \times PaceAdj

Step 5: Project Points from Field Goals

Revised Step 5 (using 2P%player):

PointsFG = (2PAproj \times 2P\%player \times 2) + (3PAproj \times 3P\%player \times 3)

Step 6: Project Points from Free Throws

PointsFT = FTAproj \times FT\%player

Step 7: Total Projected Points

TotalPointsproj = PointsFG + PointsFT

Consolidated Formula

Assuming you have a player's 2P%player readily available, here's the consolidated formula:

1. Calculate Pace Adjustment:

PaceAdj = \frac{PACE_{Opponent}}{PACE_{LeagueAvg}}

2. Calculate Projected Field Goal Attempts:

FGAproj = FGA36 \times \left( \frac{MP_{proj}}{36} \right) \times USG\%_{adj} \times DVP_{FGA\_Pos} \times PaceAdj

3. Calculate Projected Free Throw Attempts:

FTAproj = FTA36 \times \left( \frac{MP_{proj}}{36} \right) \times USG\%_{adj} \times DVP_{FTA\_Pos} \times PaceAdj

4. Calculate Total Projected Points:

TotalPointsproj = \left[ (FGA_{proj} \times (1 - 3PAr_{player})) \times 2P\%_{player} \times 2 \right] + \left[ (FGA_{proj} \times 3PAr_{player}) \times 3P\%_{player} \times 3 \right] + \left[ FTA_{proj} \times FT\%_{player} \right]

How to Use and Refine

  • Data Collection: The biggest hurdle will be consistently finding DVPFGA_Pos and DVPFTA_Pos data. Many DVP resources focus on total fantasy points or raw points allowed, rather than specific attempts allowed. You might need to derive these yourself by looking at opponent stats for each position (e.g., how many FGA opposing Point Guards average against Team X versus the league average for PGs).
  • MPproj (Projected Minutes): This is the most subjective input. Start with the player's season average, but heavily adjust for:
    • Recent game minutes (last 3-5 games).
    • Injury status of the player or teammates.
    • Coach's rotations.
    • Blowout potential (check spread and over/under).
  • USG%adj: Default to 1.0. Only change if there's a strong reason (e.g., key offensive player is injured, increasing your player's expected usage).
  • Recent Player Averages: Instead of season-long FG%player, 3P%player, FT%player, and 3PArplayer, consider using a rolling average (e.g., last 10-15 games) if the player's form has significantly changed.
  • Round to Nearest Whole Number: Once you get your TotalPointsproj, round it to the nearest whole number for comparison to prop lines.
  • Start Simple, Then Add Complexity: This formula is a good starting point. As you get comfortable, you could add factors for:
    • Individual defender match-ups.
    • Home/Away splits.
    • Rest advantage/disadvantage.
